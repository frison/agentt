## Endpoints:

*   `/health`: Returns 200 OK if the service is running.
*   `/entityTypes`: Returns a JSON array describing the configured entity types (names, descriptions, paths).
*   `/discover/{entityType}`: Returns a JSON array of discovered and valid entities of the specified type.
    *   Supports filtering by any frontmatter key present in the discovered entities using query parameters (e.g., `?tag=git&priority=high`).
    *   Behavior-specific: Use `?tier=must` or `?tier=should` to filter behaviors.

Use the information provided by `/entityTypes` and the `/discover` endpoints to retrieve relevant guidance before proceeding with tasks.
